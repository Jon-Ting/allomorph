# Repo Skill: Run core repository checks

This skill is intended for AI agents to execute the repository's main validation workflow and report the results.

## When to use

- Run the project's tests and summarize failures.
- Check lint results and report rule violations.
- Build the MkDocs documentation and confirm success.
- Verify the `allomorph` Python package imports correctly.

## Recommended commands

```bash
# Install development dependencies if needed
uv venv && uv pip install -e ".[dev]"

# Run tests
pytest -q

# Lint
ruff . --show-fixes

# Build docs
mkdocs build -d site

# CLI smoke test
python -c "import allomorph; print('OK' if hasattr(allomorph, '__name__') else 'LOAD_FAIL')"
```

## Expected output

- **Tests:** `All tests passed` or failing test names plus summaries.
- **Linter:** `No linter errors` or a list of rule violations.
- **Docs:** `Docs build succeeded` and the generated `site/` directory, or build errors.
- **CLI smoke:** `OK` or a detailed import failure.

## Notes

- Do not change source files while running checks; report results only.
- If requested to fix issues, do so in a follow-up change with tests.
