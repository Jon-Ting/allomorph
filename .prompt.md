Prompt: run-tests-lint-docs

Purpose: Run the project's core checks (tests, linter, docs build) and return a concise summary of failures or confirmation of success.

Steps for the agent to perform
1. Run the test suite and report failing tests (paths and error messages).
2. Run the linter and report any errors/warnings (file, rule, line).
3. Build the docs and report whether build succeeded and the build output location.
4. Run a quick CLI smoke test to ensure the `allomorph` entrypoint loads.

Recommended commands

```bash
# Activate environment (if needed) and run tests
uv venv && uv pip install -e ".[dev]"
pytest -q

# Lint
ruff . --fix --show-fixes

# Build docs
mkdocs build -d site

# CLI smoke test
python -c "import allomorph; print('OK' if hasattr(allomorph, '__name__') else 'LOAD_FAIL')"
```

Expected agent output
- **Tests:** list of failing test files and tracebacks, or `All tests passed`.
- **Linter:** list of rule violations or `No linter errors`.
- **Docs:** `Docs build succeeded` and `site/` path, or build errors.
- **CLI smoke:** `OK` or `LOAD_FAIL` plus exception if present.

Notes
- Do not modify source files while running these checks; only run and report. If fixes are requested, make them in a separate change with tests.
